---
title: 'Update Task'
api: 'PUT /api/tasks/{taskId}'
description: 'Update an existing task'
---

Update one or more fields of an existing task. All fields are optional - only include the fields you want to update.

## Path Parameters

<ParamField path="taskId" type="number" required>
  The unique identifier of the task to update
</ParamField>

## Request Body

All fields are optional. Only include the fields you want to update.

<ParamField body="title" type="string">
  The title of the task
</ParamField>

<ParamField body="description" type="string">
  Description of the task
</ParamField>

<ParamField body="folderId" type="number">
  ID of the folder to assign the task to
</ParamField>

<ParamField body="tagIds" type="array">
  Array of tag IDs to associate with the task (replaces existing tags)
</ParamField>

<ParamField body="durationTagId" type="number">
  ID of the duration tag for time estimation
</ParamField>

<ParamField body="priority" type="number">
  Priority level of the task (higher numbers = higher priority)
</ParamField>

<ParamField body="status" type="string">
  Status of the task. Valid values: `pending`, `in_progress`, `completed`, `cancelled`
</ParamField>

<ParamField body="dueDate" type="string">
  Due date in ISO 8601 format (YYYY-MM-DD)
</ParamField>

<ParamField body="dueTime" type="string">
  Due time in ISO 8601 format (HH:mm:ss)
</ParamField>

<ParamField body="displayOrder" type="number">
  Display order for sorting tasks
</ParamField>

<ParamField body="isArchived" type="boolean">
  Whether the task should be archived
</ParamField>

## Response

<ResponseField name="taskId" type="number">
  Unique identifier for the task

<ResponseField name="folderId" type="number">

  Timestamp when the task was last updated

  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "status": "completed"
  }'

  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "isArchived": true
  }'
